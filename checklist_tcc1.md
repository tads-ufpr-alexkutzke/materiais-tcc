# ✅ Checklist de Entregas — TCC 1

> Marque os itens conforme forem sendo concluídos. Abrange todos os entregáveis obrigatórios conforme a **Res. nº 01/2025 - TADS** e as diretrizes da disciplina **DS950**.

---

## 📄 ARTIGO CIENTÍFICO (IEEE, 6–8 páginas, até 2 figuras)

### 1. Introdução
- [ ] Problema de pesquisa e pergunta norteadora definidos
- [ ] Objetivo geral e objetivos específicos redigidos (⚠️ não são requisitos de software!)
- [ ] Motivação e justificativa do trabalho

### 2. Fundamentação Teórica
- [ ] Conceitos fundamentais para entender o problema
- [ ] Revisão de soluções semelhantes / trabalhos relacionados
- [ ] Diferenciais do projeto proposto em relação ao estado da arte
- [ ] Mínimo de 5–10 referências acadêmicas relevantes

### 3. Materiais e Métodos
- [ ] Metodologia de desenvolvimento definida e descrita
- [ ] Diagrama da arquitetura da solução
- [ ] Modelagem do sistema descrita (diagramas citados aqui, detalhes nos apêndices)
- [ ] Tecnologias e ferramentas utilizadas

### 4. Considerações Finais
- [ ] Retomada do problema e objetivos
- [ ] Síntese da solução proposta
- [ ] Resultados esperados no TCC 2

### Qualidade do Artigo
- [ ] Resumo e Abstract (150–500 palavras, parágrafo único)
- [ ] Palavras-chave e Keywords (3 a 5)
- [ ] Formatação IEEE duas colunas
- [ ] Entre 6 e 8 páginas
- [ ] Máximo de 2 figuras
- [ ] Referências formatadas (ABNT)
- [ ] Revisão ortográfica e gramatical
- [ ] Nenhuma referência de blog ou Wikipedia

---

## 🧩 ESPECIFICAÇÃO TÉCNICA DO SISTEMA

### Estrutura Geral
- [ ] **Capa** — nome do sistema, autores, orientador, data
- [ ] **Termo de Aprovação** — página para assinatura posterior
- [ ] **Resumo** — mínimo 150 palavras
- [ ] **Abstract** — versão em inglês
- [ ] **Sumário**
- [ ] **1. Contextualização** — 1 a 3 páginas descrevendo o sistema e cada diagrama
- [ ] Pelo menos 3 referências na Contextualização

### Apêndices
- [ ] **Apêndice A — Diagrama de Casos de Uso** (atores, casos, relacionamentos)
- [ ] **Apêndice B — Diagrama de Classes** (entidades, atributos, métodos, multiplicidades)
- [ ] **Apêndice C — Histórias de Usuário + Protótipos de Tela + Diagramas de Sequência** (mín. 3 casos)
- [ ] **Apêndice D — Modelo Físico/Lógico do Banco de Dados**
- [ ] **Apêndice E — Especificação de APIs** (endpoints, métodos, payloads, respostas)
- [ ] **Apêndice F — Modelos de IA** (se aplicável)

## ⚙️ PROVA DE CONCEITO (PoC)

- [ ] Funcionalidade implementada e testada
- [ ] Integração frontend/backend funcional
- [ ] Banco de dados operacional com dados de exemplo
- [ ] Demonstração executável (local ou deploy)
- [ ] Código fonte versionado no repositório
- [ ] LEIA-ME com instruções de instalação, configuração e execução
- [ ] Licença GPLv3 incluída
- [ ] Licenças de terceiros verificadas (compatíveis com GPLv3)
- [ ] Conformidade com LGPD verificada (se aplicável)

---

## 📦 REPOSITÓRIO

- [ ] Código fonte completo no repositório
- [ ] LEIA-ME com: descrição, requisitos, instruções de instalação, tecnologias
- [ ] Licença GPLv3 configurada
- [ ] Artigo em PDF disponível
- [ ] Especificação Técnica em PDF disponível
- [ ] Links funcionais e acessíveis ao orientador

## 🎤 APRESENTAÇÃO / AVALIAÇÃO

- [ ] Prova de conceito pronta para demonstração ao vivo
- [ ] Roteiro de apresentação (conforme orientador)
- [ ] Slides de apoio preparados (se necessário)
- [ ] Ensaio da apresentação realizado
- [ ] Correções pós-banca realizadas (se houver)
